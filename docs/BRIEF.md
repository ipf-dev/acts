# Project Brief

`acts`는 하모니 힐즈 IP 콘텐츠를 더 빠르고 일관되게 제작하기 위한
AI 기반 콘텐츠 제작 및 관리 플랫폼이다. 이 문서는 왜 지금 이 제품이
필요한지, 무엇을 먼저 만들어야 하는지, 어떤 기준으로 범위를 줄일지를
정리한다.

## Product Snapshot

- Product name: `acts`
- Category: 생산성 / 업무
- Current stage: MVP definition
- One-line summary: 하모니 힐즈 IP 세계관과 교육 철학을 유지하면서 시나리오, 이미지, 제작 자산을 통합 관리하고 AI로 콘텐츠 생산 속도를 높이는 지능형 제작 스튜디오
- Target IP: `Harmony Hills`
- Primary delivery milestone: `2026-04-01`

## Planning Intent

- Why this project exists now: 프로젝트가 커지면서 팀별 자산 관리가 분산되어 중복 제작, 검색 비용, 협업 지연, 톤앤매너 붕괴 위험이 커지고 있다.
- What decision this brief should unlock: 제한된 인력과 기한 안에서 어떤 기능을 1차 MVP로 고정할지 결정한다.
- What "good enough" means for the first release: 내부 제작자가 하나의 플랫폼에서 세계관 가이드를 참고하며 시나리오 초안과 이미지 초안을 생성하고, 그 결과물을 저장·검색·이력 추적할 수 있으면 충분하다.

## Primary User

- Primary user: 하모니 힐즈 프로젝트에 참여하는 내부 콘텐츠 제작자
- User groups: 콘텐츠팀, 마케팅팀, 영상팀, 디자이너팀
- Current workflow or environment: 각 부서가 시나리오, 디자인, 영상 자산을 서로 다른 방식과 도구로 개별 관리한다.
- Biggest pain point: 필요한 애셋과 최신 맥락을 찾기 어렵고, 작업 현황과 수정 이력이 분산되어 재사용과 협업이 느리다.
- Why current alternatives are not enough: 범용 CMS나 일반 AI 도구는 하모니 힐즈의 세계관, 캐릭터 관계, 레벨별 커리큘럼, SEL 철학을 내재화하지 못한다.

## Problem Statement

하모니 힐즈 콘텐츠 제작은 시나리오, 디자인, 영상 등 여러 부서가 동시에
관여하는데, 현재는 자산이 팀별로 파편화되어 있어 검색과 재활용이 어렵다.
이 때문에 부서 간 작업 현황 파악이 늦고 동일하거나 유사한 애셋을 반복 제작하는
비효율이 발생한다. 더 큰 문제는 팀마다 다른 작업 방식과 기준이 누적되며
하모니 힐즈의 핵심 가치와 톤앤매너가 흔들리고, 장기적으로 세계관 붕괴 위험이
커진다는 점이다.

## First-Slice Goal

- User can: 특정 캐릭터와 상황을 입력해 하모니 힐즈 가이드에 맞는 시나리오 초안과 이미지 초안을 생성하고 저장할 수 있다.
- System provides: 통합 자산 관리 허브, 검색 가능한 자산 목록, 생성 결과 저장, 기본 수정 이력, 세계관 가이드 기반 AI 생성 진입점
- Team can demo: 기획자가 "한복 입고 축제 가는 토니" 같은 프롬프트로 시나리오와 이미지를 만들고, 결과물이 자산 허브에 누적되어 다른 팀이 검색·재사용하는 흐름을 보여줄 수 있다.

## Scope

### In Scope

- `@iportfolio.co.kr` 도메인으로 제한된 Google SSO 로그인
- Google SSO 로그인 성공 이벤트 로깅과 실패 사유 알림
- PostgreSQL 기반 사용자 디렉터리, 단일 조직 마스터, 관리자 감사 로그
- 로그인 사용자에 대한 조직 상태 확인과 관리자 수동 지정
- 전사 열람자 이메일 allowlist와 변경 즉시 권한 재계산
- 로그인 성공, 조직 변경, allowlist 변경에 대한 DB 감사 로그 저장 및 조회
- LocalStack S3 기반 자산 업로드 모달과 자산 라이브러리 화면
- 태그 입력, 설명 입력, 파일명 기반 자동 태그 제안, 이미지 해상도 추출, 제작자/조직 메타데이터 저장
- PostgreSQL 자산 카탈로그(`assets`, `asset_files`, `asset_tags`, `asset_events`) 기반 검색과 이력 확장 구조
- 자산 상세 조회 모달과 다운로드, 메타데이터 수정, 기본 변경 이력/메타데이터 탭
- 현재 구현된 자산/관리자 화면에 대해 Figma Make 기준 레이아웃과 시각 톤 반영
- 하모니 힐즈 IP 가이드와 교육 철학을 반영한 AI 시나리오 초안 생성
- 하모니 힐즈 세계관 스타일에 맞춘 AI 이미지 생성 진입점
- 생성 및 업로드 자산의 통합 저장, 목록 조회, 검색
- 자산별 수정 이력과 생성 이력 기록
- 내부 팀이 공통으로 참고할 수 있는 세계관 및 톤앤매너 가이드 접근
- 부서 간 재사용을 위한 기본 메타데이터 구조

### Out of Scope

- 외부 고객 또는 학습자 대상 공개 서비스
- Google 외 다른 SSO 제공자 지원
- 첫 릴리즈에서의 조직 자동 매핑
- 고도화된 승인 워크플로우와 복잡한 권한 체계
- 정교한 학습 분석 대시보드나 운영 통계
- 모든 제작 도구를 완전히 대체하는 대규모 DCC 통합
- 첫 릴리즈에서의 완전 자동 생성 파이프라인

## Core User Flow

1. 콘텐츠 기획자가 ACTS에 접속해 새 콘텐츠 작업을 시작한다.
2. 기획자는 캐릭터, 레벨, 상황, 교육 의도 등을 입력해 AI 시나리오 봇에게 초안을 요청한다.
3. 시스템은 하모니 힐즈의 세계관과 교육 철학에 맞춘 시나리오 초안을 생성한다.
4. 사용자는 초안을 수정·보완하고, 같은 맥락으로 AI 이미지 생성을 요청한다.
5. 시스템은 세계관에 맞는 스타일의 이미지를 생성하고 시나리오와 함께 자산 허브에 저장한다.
6. 관련 팀원은 저장된 자산을 검색하고 재활용하거나 수정 이력을 확인한다.

## Constraints

- Team constraints: 개발 인력 1명 기준으로 핵심 흐름만 우선 구현해야 한다.
- Technical constraints: Gemini 및 Nano Banana 2 연동 과정에서 예기치 않은 기술 문제가 발생할 수 있다.
- Delivery constraints: 2026년 4월 1일까지 데모 가능한 수준의 MVP를 완성해야 한다.
- Data, security, or compliance constraints: 초기에는 내부 제작 자산과 IP 가이드를 안전하게 다뤄야 하며, AI 품질은 학습 데이터의 충분성과 정확도에 크게 의존한다.

## Success Signal

- Primary success signal: 내부 제작자가 ACTS를 통해 기존 대비 더 빠르게 초안을 만들고, 저장된 자산을 실제 후속 작업에 재사용한다.
- Observable demo or launch condition: 하나의 콘텐츠 주제에 대해 시나리오 생성, 이미지 생성, 자산 저장, 검색, 이력 확인이 하나의 흐름으로 동작한다.
- Failure signal to watch for: 생성 품질 불안정으로 결과물이 재사용되지 않거나, 팀이 기존 방식으로 되돌아가 플랫폼 사용률이 낮아진다.

## Differentiation

ACTS의 핵심 차별점은 범용 CMS나 일반 생성형 AI 도구가 아니라, 하모니 힐즈의
독자적인 세계관과 교육 철학을 학습하고 내재화한 맞춤형 AI 제작 스튜디오라는
점이다. 목표는 "누가 작업하더라도 하모니 힐즈답게" 제작할 수 있는 환경을
만드는 것이며, 이를 통해 경쟁사가 쉽게 복제할 수 없는 IP 기반의 생산 효율성과
일관성을 확보한다.
