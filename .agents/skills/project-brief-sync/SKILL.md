# Project Brief Sync

## Workflow

1. Read `docs/BRIEF.md`.
2. Read `docs/ARCHITECTURE.md`.
3. Update product scope first, then structural boundaries.
4. Remove stale promises before adding new ones.
5. Keep output concise and operational.

## Default Output

- updated scope summary
- updated architecture note if boundaries changed
- short list of unresolved decisions
