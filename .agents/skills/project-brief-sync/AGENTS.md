# Project Brief Sync

Use this skill when the product story and the repo shape risk drifting apart.

## Guardrails

- Update `docs/BRIEF.md` before code when the requested change alters user value.
- Update `docs/ARCHITECTURE.md` when the requested change introduces a new boundary, app, package, or integration.
- Prefer deleting stale statements over layering contradictory notes.

## Review Order

1. Context and problem
2. first-slice goal
3. non-goals
4. constraints
5. structure and boundaries
